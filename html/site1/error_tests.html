Test 400 - Bad Request

Envoie une requête HTTP syntaxiquement mal formée (par exemple, sans le protocole obligatoire HTTP/1.1).
Note : Les navigateurs standards formatent correctement leurs requêtes, utilisez le terminal :

printf "GET / \r\n\r\nHost: localhost" | nc 127.0.0.1 8080

Test 408 - Request Timeout

Exécute une requête incomplète avec une pause de 16 secondes (fichier config client timeout = 15).
Note : Ce test simulant un client lent (Slow Loris) ne peut pas être fait via le navigateur.

Copiez-collez cette commande dans votre terminal :

(printf "POST / HTTP/1.1\r\nHost: 127.0.0.1:8080\r\nContent-Length: 100\r\n\r\nCoucou..."; sleep 16; printf " Fin\r\n") | nc 127.0.0.1 8080

Test 413 - Payload Too Large

Envoie une requête POST avec un corps de 1 048 577 octets, dépassant la limite configurée de client_max_body_size (1 048 576 octets).
Note : Ce test nécessite l'utilisation de curl pour générer un fichier de la taille requise.

Copiez-collez cette commande dans votre terminal :

curl -X POST --data-binary @<(dd if=/dev/zero bs=1048577 count=1 2>/dev/null) http://127.0.0.1:8080/uploads/large_file.tmp -i

Test 414 - URI Too Long

Envoie une requête avec une URI de plus de 8 Ko, dépassant la limite de buffer du serveur pour la ligne de requête.
Note : Ce test utilise une commande pour générer une URL très longue.

Copiez-collez cette commande dans votre terminal :

curl "http://127.0.0.1:8080/?q=$(python -c "print('a'*8192)")" -i

Test 505 - HTTP Version Not Supported

Envoie une requête en HTTP/1.0 au serveur, alors que celui-ci attend HTTP/1.1.
Ce test permet de vérifier que le serveur rejette une version HTTP non supportée avec une erreur 505.

Copiez-collez cette commande dans votre terminal :

curl --http1.0 http://127.0.0.1:8080/ -i
